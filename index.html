The scope of var, let, and const


Although var, let, and const may seem similar but are different in functionalities, here they are:


  • var declarations are globally scoped whereas let and const are block scoped.
  • var variables can be updated and re-declared.
    In case of, let the variables can be updated but not re-declared whereas const variables can neither be updated nor re-declared.
  • They are all hoisted to the top of their scope. But while var variables are initialized with undefined, let and const variables are not initialized.
  • While var and let can be declared without being initialized, const must be initialized during declaration.

    • The Use Cases of Null and Undefined:


    JavaScript undefined

    If a variable is declared but the value is not assigned, then the value of that variable will be undefined.
    For example,

    let name;
    console.log(name); // undefined
    It is also possible to explicitly assign undefined to a variable.
    For example,

    let name = "Felix";
    // assigning undefined to the name variable
    name = undefined
    console.log(name); // returns undefined



    JavaScript Null

    In JavaScript, null is a special value that represents an empty or unknown value.
    For example,

    let number = null;

    The code above suggests that the number variable is empty at the moment and may have a value later.

    Note: null is not the same as NULL or Null.
